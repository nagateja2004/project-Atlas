# Data provenance

## Source corpus

Every record under `data/synthetic_epc/` was created for the Project Atlas demonstration. The project name, companies, people, vendors, equipment, dates, clauses, prices, delays, RFIs, answers, procedures, and acceptance criteria are fictional. No customer, employer, supplier, shipment, or site data is represented.

The corpus uses realistic EPC language but does not reproduce or claim official requirements from TIA-942, BICSI, Uptime Institute, UL, AHRI, or another standards body. A standards name appearing in fictional vendor evidence is not a certification or authoritative requirement.

## Processing lineage

Original files are uploaded to project/document paths and recorded in PostgreSQL with a content hash. Parsing produces pages, metadata, persisted parents, `original_text`, and deterministic `contextual_text`. Original text is used for quotations and citations; contextual text is used for embedding and lexical retrieval. Each Qdrant payload includes `project_id`, document/chunk identifiers, page, section, revision/approval metadata, and index version.

Gemini may rewrite queries, extract structured values, explain verified calculations, or generate evidence-grounded answers when `ATLAS_GEMINI_API_KEY` is configured. Deterministic evaluation uses local test doubles where documented; it is not evidence of live-model latency or quality.

## Derived outputs

- `evaluation/latest.json` and `evaluation/latest.md` are generated by `python3 -m evaluation.run_all`.
- `reports/rag_evaluation.json` and `.md` are generated by the comparative RAG harness.
- PostgreSQL findings, decisions, test records, NCRs, mitigations, and audit events are derived from the seeded synthetic records and user actions.
- NetworkX JSON is a derived project visualization, not a governed master-data source.

No manual-effort measurements are supplied. Missing measurements remain `NOT_MEASURED`; hours saved must not be inferred.
